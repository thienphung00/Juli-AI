## What this skill is for

`to-issues` converts a plan/spec/PRD into a set of **independently-grabbable GitHub issues** using tracer-bullet **vertical slices**.

## What “vertical slice” means here

- A thin, end-to-end path through all relevant layers
- Verifiable on its own (demoable or testable)
- Minimal coupling to other slices

## HITL vs AFK

- **AFK**: can be implemented and merged without waiting on human decisions
- **HITL**: requires a human checkpoint (architecture/product/design/approval)
