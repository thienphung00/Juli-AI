# Design System: Vercel Geist Dashboard

**Project ID:** vercel-geist-dashboard-light  
**Sources:** [vercel.com/design.md](https://vercel.com/design.md), [vercel.com/geist](https://vercel.com/geist/introduction), Mobbin dashboard screenshots (Overview, All Projects, Analytics, Speed Insights)

## 1. Visual Theme & Atmosphere

Geist is Vercel's design language for developer infrastructure — precise, restrained, and engineered to disappear behind the data. The dashboard aesthetic reads as a **high-contrast monochrome workspace**: near-white canvases, ink-black typography, and hairline borders that organize dense operational information without visual noise. Color is treated as punctuation, not decoration; blue appears on charts and links, green on ready states, red on failures, and amber on thresholds — never as ambient brand wash.

The mood is **calm authority**. Whitespace is generous but never wasteful: cards breathe with 24px internal padding, sections separate by 32–40px, and the persistent left sidebar anchors navigation while the main canvas stays open for metrics, deployment previews, and time-series charts. Surfaces stack through tonal difference (`background-100` vs `background-200`) and translucent `gray-alpha` borders rather than heavy shadows. The overall feel is that of a precision instrument panel — every pixel earns its place, numbers align in tabular mono, and the single black primary button signals the one action that matters on a view.

## 3. Typography Rules

### Hierarchy & Weights

**Geist Sans** carries all UI prose — geometric, neutral, with negative letter-spacing on headings that tightens as size grows (display voice without exceeding weight 600). **Geist Mono** appears for commit hashes, deployment URLs, route paths, and tabular numerals in tables and metrics.

| Token | Size / Weight | Dashboard Usage |
|---|---|---|
| `heading-32` / `stat-lg` | 32px / 600 | Large metric values (Visitors: 13, Real Experience Score: 72) |
| `heading-24` | 24px / 600 | Section titles inside cards ("Production Deployment") |
| `heading-16` | 16px / 600 | Card headers ("Observability", "Analytics", "Usage") |
| `heading-14` | 14px / 600 | Table column headers, tab labels |
| `label-14` | 14px / 400 | Sidebar nav items, dropdown labels, form fields |
| `label-14-strong` | 14px / 500 | Project names on cards, filter button text |
| `label-13` | 13px / 400 | Secondary metadata (branch names, "6h" time windows) |
| `label-12` | 12px / 400 | Badge text ("Pro Trial"), chart axis labels, legal/footer |
| `label-13-mono` | 13px mono | Commit SHAs, deployment URLs, route paths (`/`) |
| `button-14` | 14px / 500 | Standard button labels ("Repository", "Visit") |
| `button-16` | 16px / 500 | Large CTAs ("Install Extension", "Add New…") |
| `copy-14` | 14px / 400 | Descriptive body in modals and empty states |
| `copy-13` | 13px / 400 | Helper text, performance descriptions |

### Spacing Principles

Headings use tight tracking (`-0.28px` to `-1.28px`) for a compressed, engineered voice. Body and labels use normal tracking. Line-height stays compact on labels (1.43×) and more generous on copy (1.5×). Stat numbers always use tabular figures when available. Title Case for labels, buttons, tabs, and card titles; sentence case for descriptions and helper text.

## 4. Component Stylings

### Buttons

Three tiers dominate the dashboard:

- **Primary (Ink):** Solid `#171717` fill, white text, 6px radius, 40px height (48px for hero CTAs). Used for "Add New…", "Install Extension" — one per view maximum.
- **Secondary:** White fill, `gray-alpha-400` hairline border, ink text. Used for "Repository", "Instant Rollback", "Visit", "Billing".
- **Tertiary/Ghost:** Transparent fill, ink text, hover tint via `gray-alpha-100`. Used for "Preview", "Source", inline actions.

Hover steps fill from `100` → `200` → `300` on the gray scale; focus shows a double-ring: 2px white gap + 2px `#006bff` ring. Disabled: `gray-100` fill, `gray-700` text.

### Cards & Dashboard Containers

Cards are the primary content unit. White (`background-100`) fill, 1px `gray-400` or `gray-alpha-400` border, **6px radius**, **24px padding** (16px compact, 32px hero). Elevation is minimal — `0 2px 2px rgba(0,0,0,0.04)` only when a card needs lift. No colored card backgrounds except state tints (blue checklist completion, red alert callouts).

**Project cards** (All Projects grid): 2-column layout, project icon + name + URL, GitHub repo link, latest commit one-liner, date + branch, status icon (green check / red exclamation / spinner). **Metric cards** (Analytics top row): horizontal tab strip with active indicator = 2px bottom border in ink; large stat number centered.

**Production Deployment card:** Split layout — left preview iframe thumbnail (~40% width), right metadata stack (Deployment URL, Domains, Status dot, Created by, Source commit). Footer link row with recommendation badge pill.

### Navigation

**Fixed left sidebar** (~240px): `background-200` fill, full viewport height. Structure top-to-bottom:

1. Team/account switcher with badge pill ("Pro Trial")
2. Search input with magnifier icon and `F` keyboard shortcut chip
3. Primary nav list (Overview, Deployments, Logs, Analytics, Speed Insights…) — icon + `label-14`, active item gets `gray-100` rounded pill background
4. Expandable sections with chevron (Observability, Firewall, CDN)
5. Secondary nav (Domains, Integrations, Storage…)
6. Bottom: Usage, Support, Settings
7. User profile row with avatar, name, settings + notification bell

**Top bar:** Project selector dropdown, centered or left page title, right-aligned filter dropdowns (Environment: Production, Date: Last 7 Days) and action buttons.

### Inputs & Forms

Search fields: white fill, hairline border, 6px radius, 40px height, placeholder in `gray-700`. Segmented toggles (Desktop / Mobile): pill track in `gray-100`, active segment white with shadow. Dropdowns: chevron icon, same border treatment as inputs. Badge counts (e.g., "1" on Environment Variables) use amber/yellow fill with dark text.

### Domain-Specific Components

**Line / area charts:** Blue stroke (`#006bff`), light blue gradient fill under curve, dashed projection for incomplete days, vertical crosshair + tooltip card on hover. Y-axis labels in `label-12`, X-axis dates abbreviated ("Apr 22", "Yesterday").

**Data tables with tabs:** Underline tab bar (Pages / Routes / Hostnames; Referrers / UTM Parameters). Rows: primary label left, numeric count right-aligned in mono. Favicon/icon per referrer row.

**Core Web Vitals panel:** Left column of metric rows — label, value, horizontal status bar (green/amber/red). Circular gauge for Real Experience Score with color ring matching threshold. Route breakdown table in three columns: Poor (<50), Needs Improvement (50–90), Great (>90).

**Status indicators:** 8px filled circle — green (Ready), red (Error), animated spinner (Building). Badge pills for counts ("4 Recommendations" in blue tint).

**Alert callout:** Light red border + background, warning icon, single action button ("Update Projects").

**Popover (Visit with Toolbar):** White surface, popover shadow stack, QR code centered with Vercel logo, instructional copy, black CTA at bottom.

**Progress bars:** Thin horizontal track in `gray-200`, fill in blue (`blue-700`) for usage/credit consumption.

**Checklist rows:** Unchecked = plain row; checked = blue checkmark icon + light blue row background.

## 5. Layout Principles

### Grid & Structure

Dashboard uses a **sidebar + fluid main** shell. Main content centers in a **1200px max column** with responsive side padding. Overview pages use a **card grid**: full-width hero card on top, then 3-column widget row (Checklist | Observability | Analytics), then full-width sections below (Active Branches). All Projects uses **asymmetric 2-column**: narrow left widgets (Usage, Alerts, Previews) + wide 2-column project grid.

Analytics and Speed Insights use **metric header row** + **full-width chart** + **2-column table pair** at bottom.

### Whitespace Strategy

Base unit: **4px**. Rhythm: 8px inside groups, 16px between groups, 32–40px between sections. Card internal padding: 24px default. Sidebar item gap: 2–4px vertical. Chart area padding: 16px top/bottom, 24px sides.

### Alignment & Visual Balance

Left-aligned text throughout; numbers right-aligned in tables. Icons consistently 16–20px, vertically centered with labels. Status dots and badges sit at row trailing edge. Deployment preview thumbnails left-aligned in split cards. Filter controls right-aligned in page header.

### Responsive Behavior & Touch

Breakpoints: `sm` 401px, `md` 601px, `lg` 961px, `xl` 1200px, `2xl` 1400px. Sidebar collapses to icon rail or drawer below `lg`. Project grid drops from 2 columns to 1. Metric cards stack vertically on mobile. Touch targets minimum 40px (buttons/inputs); large CTAs 48px. Segmented controls and tabs maintain 44px tap height.

## 6. Design System Notes for Stitch Generation

### Language to Use

Describe screens as **"developer infrastructure dashboard"**, **"monochrome precision panel"**, **"restrained high-contrast SaaS"**. Emphasize: ink-on-white typography, hairline borders over shadows, color only for state, tabular numbers, generous but structured whitespace. Avoid marketing gradients on dashboard surfaces (reserve spectral mesh for marketing heroes only).

### Color References

- Canvas: Canvas White `#ffffff`, Sidebar Mist `#fafafa`
- Text: Ink Black `#171717`, Secondary Slate `#4d4d4d`, Tertiary Muted `#8f8f8f`
- Interactive: Vercel Blue `#006bff`, Ink Black `#171717` (primary CTA)
- State: Ready Green `#28a948`, Warning Amber `#ff9300`, Error Red `#ea001d`
- Charts: Vercel Blue line `#006bff`, Chart Wash fill `#dfefff`
- Borders: Hairline Border `#eaeaea`, Translucent Divider `#00000014`

### Incremental Iteration

Start with the **sidebar shell + one card** to lock border radius and gray scale. Add **typography hierarchy** (stat number vs label-14 vs label-12). Introduce **blue only on charts and links** — resist adding blue to backgrounds. Layer **state colors last** (green dots, red bars, amber badges). Match **shadow depth** to component type: flat cards, elevated popovers only. Verify focus rings and active nav states before adding data density.
